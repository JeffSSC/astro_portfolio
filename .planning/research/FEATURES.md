# Feature Landscape: Portfolio Sites 2026

## Table Stakes (Minimum Expectations)

| Feature | Why Expected | Complexity |
|---------|------------|-----------|
| Homepage with project grid | Immediate showcase of work | Low |
| About page | Personal/career story | Low |
| Contact information | Enable hiring conversation | Low |
| Responsive design | Mobile viewers are significant | Medium |
| Theme toggle (dark/light) | User preference | Low |

## Differentiators (Valued but Not Expected)

| Feature | Value Proposition | Complexity |
|---------|-------------------|------------|
| Case study pages | Show depth, not just thumbnails | Medium |
| View Transitions | Smooth navigation feels premium | Low |
| Custom cursor | Design-forward attention to detail | Low |
| Micro-interactions | "Delight" factor | Low |

## Anti-Features (Avoid)

| Anti-Feature | Why Avoid | Alternative |
|--------------|---------|-------------|
| Auto-playing video carousels | Performance killer, UX annoyance | Static thumbnails with click-to-expand |
| Hero sliders | Outdated, distracts from content | Single strong hero with clear CTA |
| Excessive particles/Three.js | Slow loads, not maintainable | CSS grid backgrounds |
| Too many projects | Dilutes message, decision fatigue | 3-5 strongest projects only |

## Project Recommendation for This Portfolio

The reference design includes:
- Hero (name, title, tagline)
- About section
- Experience timeline
- Tech stack grid
- Projects grid
- Testimonials
- Contact info
- Footer

**This is the correct scope** — matching 2026 best practices of restraint and focus.
